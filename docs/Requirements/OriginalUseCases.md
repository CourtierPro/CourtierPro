# Original Use Cases (From Design Documents)

## Broker
- Broker prioritizes transactions
- Broker assigns transactions
- Broker exports transaction summary (CSV)
- Broker views all transactions dashboard
- Broker filters transactions
- Broker sorts transactions
- Broker updates transaction stage
- Broker enters transaction dates & milestones
- Broker edits transaction dates & milestones
- Broker views calculated tax breakdown
- Broker receives real-time notifications
- Broker views audit trail for transaction
- Broker runs commission tax calculation (GST/QST)
- Broker inputs commission amount
- Broker views transaction details

## Client
- Client authenticates / logs in
- Client accesses transaction summary (name/status only)
- Client views transaction timeline with % complete
- Client views own transaction progress
- Client confirms milestone completion (Acknowledge / Dispute)

## Admin
- Admin configures transaction stage definitions
- Admin configures dashboard defaults & reporting periods
- Admin views system audit logs
- Admin manages language settings
- Admin manages notification templates & delivery rules
- Admin archives old transactions
- Admin purges old transactions
- Admin creates user accounts
- Admin manages user roles
- Admin configures role-based permissions
- Admin manages system tax settings (GST/QST rates)

## Authentication (All Roles)
- User authenticates / logs in
- User resets password
